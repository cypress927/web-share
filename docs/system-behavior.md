# 系统行为文档

## 目的

本文档用于描述 `web-share` 当前版本的系统行为、状态边界和用户可见规则。

它关注的是“系统现在实际上怎么工作”，而不是未来规划。

## 1. 进程与架构

系统采用单管理器架构。

### 组成

- `run-manager`
  - 后台 HTTP 管理器
- `tray`
  - Windows 托盘入口
- `enqueue`
  - 右键菜单或命令行投递分享请求

### 运行规则

- 管理器固定监听 `21910`
- 托盘和管理器可按需启动
- 双击 `web-share.exe` 时，会在后台优先启动管理器和托盘
- 双击正常启动不会自动打开浏览器
- 双击正常启动时，如果右键菜单缺失，会自动补装
- 首次双击启动时，默认语言会跟随 Windows 系统语言初始化
- `enqueue` 会先检测管理器是否可达
- 如果管理器未启动，会自动在后台拉起管理器
- 如果托盘未启动，也会自动在后台拉起托盘
- 右键分享不会自动打开管理页

### 当前内部结构

当前实现已经分成“入口层、骨架编排层、平台实现层”三部分：

- 入口层：
  - `internal/app`
  - `internal/manager`
  - `internal/tray`
- 骨架编排层：
  - `internal/systemstate`
- 平台实现层：
  - `internal/systemstate/windows_ports_windows.go`
  - `internal/shell`

其中：

- 入口层负责接收 CLI、托盘点击和 Web 页面动作
- `internal/systemstate` 负责系统状态检查、目标态收敛、warning/error 聚合
- Windows 具体行为实现仍然放在平台实现层

### 当前系统状态编排

当前以下系统对象已经主要通过统一编排层处理：

- 右键菜单
- 自启动
- 托盘
- manager 进程
- 程序停止请求

统一编排层提供的核心语义是：

- `Ensure*`
  - 把目标收敛到某个状态
- `Snapshot`
  - 汇总当前系统状态

这意味着当前系统集成动作已经主要是“目标状态驱动”，而不是“入口各自直接硬编码判断和执行”。

## 2. 分享类型

当前支持：

- 文件分享
- 文件夹分享
- 剪贴板文本分享（快照）
- 剪贴板图片分享（快照）

### 文件分享

- 始终只读
- 页面提供文件下载
- 支持断点续传

### 文件夹分享

- 默认只读
- 设置上传密码后允许上传
- 支持子目录浏览
- 支持根目录打包下载
- 支持子文件夹单独打包下载
- 支持单文件上传
- 支持文件夹上传
- 上传使用顺序分片方案

### 剪贴板文本分享

- 创建时保存文本快照（内存态）
- 始终只读
- 支持页面查看与下载
- 标题默认由文本首行摘要生成，过长截断

### 剪贴板图片分享

- 创建时保存 PNG 快照（内存态）
- 始终只读
- 支持页面预览与下载
- 标题默认包含截图时间

## 3. 分享生命周期

### 创建

- 新建分享时记录目标绝对路径
- 文件夹分享可附带上传密码
- 新建分享默认首页隐藏
- 新建分享默认生成短分享码
- 托盘剪贴板分享会优先处理文件/文件夹复制列表，批量创建路径型分享
- 剪贴板文本/图片分享不依赖文件系统路径

### 运行期

- 路径型分享（文件/文件夹）不是快照，按文件系统实时读取
- 剪贴板文本/图片分享是创建瞬间快照，不随系统剪贴板后续变化

### 停止

- 分享可以在管理页单独停止
- 托盘退出程序时会关闭管理器和分享

## 4. 可见性规则

### 首页

- 首页只显示 `Visible=true` 的分享
- 即使首页隐藏，仍然可以通过分享码或直链访问
- 对可预览内容会直接展示卡片预览：
  - 文本摘要（支持一键复制）
  - 图片缩略图
  - 单文件信息和一键下载
  - 一键复制复制完整文本，不受卡片摘要截断影响

### 管理页

- 管理页展示所有分享
- 管理页仅允许本机访问
- 管理卡片支持文本摘要和图片缩略图预览

### 预览类型与限制

- 文本预览和图片预览均基于扩展名白名单判定
- 文本预览读取上限为 `64KB`
- 文本预览会按页面再次截断：
  - 首页约 `180` 字符
  - 管理页约 `220` 字符
  - 详情页约 `16000` 字符
- 非白名单类型不会展示预览，只提供下载
- 源文件失效时，图片缩略图请求会返回 404

## 5. 下载行为

### 文件下载

- 使用流式传输
- 不会先把整个文件读入内存
- 支持：
  - `Accept-Ranges: bytes`
  - `206 Partial Content`
  - `Content-Range`
- 支持 `HEAD`

### 目录打包下载

- 根目录按钮表示下载整个分享根目录
- 列表中的子文件夹可以单独打包下载
- 打包格式为 `.zip`
- 服务端边遍历边写出 zip
- 不先落盘整个压缩包

## 6. 上传行为

### 启用条件

- 只有文件夹分享才允许上传
- 只有设置上传密码后才允许上传

### 上传方案

- 前端按顺序切片上传
- 后端按顺序把分片追加到临时文件
- 最后一个分片完成后重命名为目标文件

### 单文件上传

- 支持

### 文件夹上传

- 支持
- 通过浏览器目录选择能力拿到相对路径
- 服务端按相对路径创建子目录并写入文件

### 上传时的目录创建规则

- 允许在“当前有效目录”下创建上传目标所需的子目录
- 不允许因为共享根目录或当前浏览目录已失效，而自动重建这些失效目录

也就是说：

- 如果当前分享根目录不存在，上传会被拒绝
- 如果当前浏览目录不存在，上传会被拒绝
- 如果当前目录存在，但上传文件夹里包含新的子目录结构，允许创建这些子目录

### 文件覆盖规则

- 当前不会覆盖同名文件

## 7. 失效路径行为

### 定义

“失效”表示某个分享记录仍存在，但它指向的真实文件系统路径已经不存在，或不再满足原本类型。

### 根路径失效

如果分享根路径不存在：

- 首页中的可见分享会显示 `已失效`
- 分享页会显示：
  - `该分享对应的文件或文件夹已不存在，可能已被移动或删除。`
- 不会自动清理该分享
- 不会自动标记停止

### 当前浏览目录失效

如果用户当前进入的子目录已不存在：

- 系统不会展示纯文字错误页
- 会重定向回上一级分享页面
- 并在页面中显示错误提示

### 文件下载目标失效

如果用户页面没刷新，直接点击下载，但文件已不存在：

- 会重定向回当前分享页面
- 并显示错误提示

### 文件夹打包目标失效

如果用户点击打包下载，但目标目录已不存在：

- 会重定向回当前分享页面
- 并显示错误提示

### 上传目标失效

如果共享根目录或当前浏览目录已不存在：

- 上传会被拒绝
- 不会因为上传而自动把失效目录重新创建出来

## 8. 页面内错误提示策略

当前系统尽量避免把访问者丢到纯文字错误页。

### 优先策略

- 对页面点击产生的错误：
  - 尽量回到分享页
  - 通过 query message 展示错误

### 本地设置页与系统设置页

`/setup` 和 `/manage/settings/system` 当前已经支持异步本地操作反馈。

前端会显示三类结果：

- success
- warning
- error

其中 warning 用于表示：

- 目标状态本来已满足
- 检测到旧残留但已自动修复
- 某些对象原本已缺失，因此没有执行实际变更

这类情况不会被当成致命失败。

### 仍然可能返回纯错误响应的情况

- 非页面型请求
- 非法参数
- 浏览器或下载器直接发起的特殊请求
- 服务器内部错误

## 9. 二维码与地址

### 管理页二维码

- 二维码指向当前分享的首选访问地址
- 优先局域网 IPv4 地址
- 没有可用局域网地址时，退回本机地址

### 首页和分享码

- 首页支持输入分享码跳转
- 分享码用于缩短分享地址输入成本

## 10. Windows 集成

### 右键菜单

- 文件：
  - `只读分享`
- 文件夹：
  - `只读分享`
  - `设置上传密码后分享`

### 密码输入

- 文件夹密码分享通过程序内原生 Windows 密码输入框收集密码
- 不依赖 PowerShell 或 VBS 黑框

### 托盘

当前托盘菜单提供：

- `打开管理页面`
- `分享当前剪贴板`
- `退出程序`

普通用户的主入口是：

- 双击程序后常驻托盘
- 需要时再从托盘打开管理页

系统设置页还可以直接停止整个程序，便于移除集成后手动删除文件。

### 日志

当前系统已经引入本地日志模块：

- `internal/logx`

日志主要记录：

- 系统状态检查
- 系统集成收敛动作
- warning
- error

默认日志目录为：

- `%LOCALAPPDATA%\WebShare\logs\YYYY-MM-DD.log`

剪贴板分享补充行为：

- 优先级为：文件/文件夹列表 > 图片 > 文本
- 文件/文件夹列表会逐条创建独立分享
- 批量创建中途失败时，不会回滚已经成功创建的分享

### 自启动

- 当前内置实现使用注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
- 自启动命令会拉起 `web-share.exe start`
- 作用是自动启动管理器和托盘
- 注册表项名称默认为：`WebShare.AutoStart`

### 当前状态语义

当前系统集成操作已经趋向以下语义：

- install / enable / start
  - 目标是确保对象处于启用或运行状态
- uninstall / disable / stop
  - 目标是确保对象处于卸载、禁用或停止状态

如果目标已经满足：

- 可以返回 success 或 warning
- 不要求一定产生实际变更

如果检测到旧残留或半安装状态：

- 可以先清理再收敛
- 页面会优先显示 warning，而不是静默吞掉

### 兼容计划任务（旧脚本）

- `scripts/` 目录下保留了旧的计划任务脚本，用于兼容旧安装流程
- 默认任务名仍为：`WebShare.AutoStart`
- 安装时若任务已存在：
  - 默认报错
  - 使用安装脚本 `-Force` 可覆盖重建
- 卸载脚本在任务不存在时会直接提示并退出，不视为失败
